
Class: System.Math
Namespace: System
ลักษณะ: static class
เหตุผลที่จัดเป็น Utility Class
•	เป็น static class จึงไม่สามารถสร้าง instance ได้
•	ไม่มีการเก็บสถานะของ object (no instance fields / no state)
•	รวมชุดของ method สำหรับการคำนวณทางคณิตศาสตร์ไว้ให้เรียกใช้จากภายนอก
•	การใช้งานเรียกผ่านชื่อคลาสโดยตรง ไม่ต้อง new object
สรุป
System.Math ทำหน้าที่เป็นตัวช่วย (helper) ที่รวบรวมฟังก์ชันสำหรับการคำนวณโดยไม่มีข้อมูลภายในของตัวเองและไม่มีการเปลี่ยนแปลงสถานะของคลาส จึงเข้าลักษณะของ Utility Class


Class: System.Text.StringBuilder
Namespace: System.Text
ลักษณะ: non-static class
เหตุผลที่จัดเป็น Mutable Class
•	สามารถแก้ไขค่าภายใน object เดิมได้หลังจากสร้าง instance แล้ว
•	มี internal state ที่เปลี่ยนแปลงได้ เช่น ความยาวของข้อความและ buffer ภายใน
•	method ต่าง ๆ ทำงานโดยแก้ไข object เดิม ไม่ได้สร้าง object ใหม่
•	มี property ที่ค่าของมันเปลี่ยนตามการทำงานของ method
สรุป
System.Text.StringBuilder เป็นคลาสที่ออกแบบมาเพื่อให้แก้ไขข้อมูลข้อความภายใน instance เดิมได้โดยตรง การเรียกใช้งาน method จะเปลี่ยนสถานะของ object เดิมแทนที่จะคืนค่าเป็น instance ใหม่ จึงมีลักษณะเป็น Mutable Class


Class: System.String
Namespace: System
ลักษณะ: sealed class
เหตุผลที่จัดเป็น Immutable Class
•	ค่าใน instance ไม่สามารถเปลี่ยนแปลงได้หลังจากสร้างขึ้นมาแล้ว
•	ไม่มี method ใดที่สามารถแก้ไขข้อมูลภายใน object เดิมโดยตรง
•	การดำเนินการที่เป็นการเปลี่ยนค่า เช่น การต่อข้อความหรือแทนที่ตัวอักษร จะสร้าง object ใหม่เสมอ
•	โครงสร้างภายในเปิดให้แก้ไขไม่ได้ และไม่มี setter สำหรับเปลี่ยนค่าโดยตรง
สรุป
System.String เป็น Immutable Class เพราะเมื่อสร้าง instance แล้วค่าภายในจะคงเดิมตลอด object การทำงานที่เกี่ยวข้องกับการเปลี่ยนแปลงข้อความจะให้ผลลัพธ์เป็น instance ใหม่แทนการแก้ไขของเดิม ทำให้ไม่มีการเปลี่ยนแปลงสถานะของ object เดิมแต่อย่างใด
